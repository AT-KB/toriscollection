# 撮影手順書(Screen-capture SOP) — Phase S / 開発部
**根拠**: `toris_ceo_kettei_memo_phase_s.md` 決定4・決定5、および実地検証(2026-07-19〜20、本セッション)。
**目的**: 「7秒テスト動画ドラフト1本がPC上に存在する」(Phase S 完了定義#1)を、$0・再現可能な標準手順で達成する。
**実行**: CEO(撮影・転送・字幕)。本書は開発部が手順化したもの。

---

## 0. なぜadb/scrcpy経由を放棄したか(実地の結論)
本セッションで adb(`screenrecord`)による実機ライブ録画を複数回試み、いずれも安定しなかった。原因は**自動スリープではなく**、以下の重なり:
- **adb screenrecord は内部音声を録れない**(アプリのラジオ/BGM音が入らない)。
- ホスト型Webアプリ(Streamlit on Render)が**この回線で頻繁に再描画/再接続**し画面が白くなる。
- **Chromeの横スワイプ=タブ切替、上端=通知シェード**が adb 操作を横取りする。
- ネイティブアプリ(Capacitor)は**スプラッシュで固まる既知事象**がある。

→ **CEOが実機の内蔵画面録画で撮る**のが、音付き・安定の唯一の現実解。決定4のとおり。

## 1. 7秒テストの合格基準(撮る前に頭に入れる)
**無音・字幕のみで、7秒以内に「アプリを開く→見知らぬ鳥がいる→本物の声で鳴く」が成立すること。**
- 冒頭2秒で「見知らぬかわいい鳥」が画面にいる(ドット絵の鳥/会う瞬間のポップ)。
- 3〜7秒で「本物の鳴き声で鳴く」ことが伝わる(ラジオ再生 or 会う瞬間)。字幕例:*"Real birds. Real songs. Your own radio."*
- 交渉不能原則:静か・押し売りしない。煽り字幕・CTA連呼はしない。

## 2. 主経路(推奨・音付き)
1. **端末の言語/アプリをEN表示**にする(i18nは既定EN)。
2. 端末の**内蔵スクリーンレコーダー**を起動し、**「メディアの音を録音」= ON**(設定名は端末により「音声=メディアの音」等)。
3. アプリを開く。**ネイティブアプリがスプラッシュで固まる場合は、Chromeで `https://toris-collection.onrender.com` を開いて撮る**(見た目は同じ、音も出る)。※Renderは無料枠でスリープするので、撮影の1分前に一度開いて起こしておく。
4. 撮りたい所作を実行(§4のショット別手順)。**撮影中は縦スクロールのみ**(横スワイプはブラウザのタブ切替を誘発)。
5. 停止 → **Google Photos**にバックアップ → PCのブラウザ/アプリで取得。
6. **CapCut(無料)**で字幕付与・トリミング。7秒版/15秒版/30秒版を書き出す。

## 3. フォールバック(端末が内部音声録画に非対応の場合)
1. **映像のみ**を内蔵レコーダー(または録画中に無音でも可)で撮る。
2. 音声は、**アプリが実際に再生しているのと同一の xeno-canto 音源ファイル**を編集段階で合成する。
   - 同一音源なので "real recordings" 表記と矛盾しない(交渉不能原則4)。
   - **出典クレジット(種名/録音者/xeno-canto番号/ライセンス)は動画または概要欄に必須**。
   - 参考: HP用に作成済みの `landing/media/garden_radio.mp3`(Cardinal/Chickadee/Blue Jay/Song Sparrow/Robin、CC BY-SA/CC0)と `landing/media/garden_radio_credit.json` がそのまま流用可能。

## 4. ショット別・撮影手順(決定メモの見せたい場面)
| ショット | 手順(要点) | 合格の勘所 |
|---|---|---|
| **① 会う瞬間(最優先/最強のかわいさ)** | 庭(Garden)タブ → 「♪ Listen closely」→ 木の枝で鳥が**ぴょんぴょん**(rite_bob)→ 手前まで来て**キラッ**+「会えた!」→ 数秒後に**出会いポップアップ** | 跳ねる→会う→ポップの連続を1カットで。確率制御なので複数テイク。跳ねる待機だけでも可愛いので必ず何か撮れる |
| **② 自分だけのラジオが育つ** | ラジオ(Radio)タブ → 「🎙 Start the radio」→ 鳥チップが順に点灯し**複数羽が重なって鳴く** → 直前に新種に会っておくと「新しい声が加わりました」バナーが出る | 音が主役。BGM+複数鳥の重なりが聞こえること |
| **③ 庭の様子** | Garden/Homeタブで植物と滞在中の鳥が見える構図 | 静かな庭。ゆっくりだけ動かす |
| **④ ネットワーク図(任意)** | Networkタブ → 下スクロールでグラフ | 完全英語のノード名(既にHPで使用) |

## 5. よくある失敗と回避
- **横スワイプ厳禁**(Chromeタブ切替を誘発)。移動は縦スクロールとタブのタップのみ。
- **画面上端に触れない**(通知シェードが降りる)。撮影前に通知を消しておく。
- **Renderは事前に起こす**(コールドスタート30〜60秒)。
- **儀式の"会う"は確率**。焦らず複数テイク。跳ねる待機カットは保険として常に撮れる。
- **EN表示を確認**してから撮る。

## 6. 完了定義(このSOPのDoD)
- 7秒テスト動画ドラフトが**1本、PC上に存在**し、CEOが合否判定できる状態。
- 素材は `marketing/phase-s/素材/` に日付付きで保管(命名例: `20260721_meeting_7s_draft.mp4`)。
