## 2. 処理（タグ付け）— 誰が・頻度・保存先・基準

### 2-1. 誰が / 頻度

- **タグ付け実務**: SNS部（決定1「運用＝SNS部」）。
- **基準の設計・改訂・原則ガード監査**: 企画部（本書＝設計、四半期の語彙見直し）。
- **頻度**: **週次**。指示書§4の週次リズム（金曜レポート）に同期させ、金曜レポート提出時までに当週分の新規表現をタグ付けして台帳へ追記する。**新規表現がゼロの週は「0件」と記録するだけ**（デイリー強制・件数ノルマは設けない＝原則1）。

### 2-2. 保存先（提案パス）

| 用途 | パス案 | 形式 |
|---|---|---|
| タグ付きコーパス台帳（本体） | `marketing/phase-s/企画部/user_language/corpus.md` | 表（1行=1表現） |
| 転用の下書き置き場（ASO/キャプション候補の芽） | `marketing/phase-s/企画部/user_language/derived_candidates.md` | 表（候補と出所リンク） |

> ディレクトリ `marketing/phase-s/企画部/user_language/` を新設案とする。台帳と派生候補を分けるのは、「生表現(事実)」と「転用案(創作)」を混ぜないため（原則4の誇張混入を構造で防ぐ）。**実ファイル作成はCEO確認後。**

### 2-3. タグの基準（初期案）

各表現に **(a)感情タグ1つ以上 ＋ (b)テーマタグ1つ以上** を付ける。

**(a) 感情タグ**（初期語彙・排他ではなく複数可）

| タグ | 定義（この語が付く条件） | 例（想定される英語表現の型） |
|---|---|---|
| `delight` | 出会い・登場に対する喜び・かわいさ反応 | "so cute", "made me smile", "I love this little one" |
| `calm` | 癒し・落ち着き・BGM文脈 | "so relaxing", "I leave it playing", "quiet" |
| `surprise` | 予期せぬ登場・想定外の発見 | "I wasn't expecting", "just showed up", "didn't know" |
| `curiosity` | 学び・生態への関心（"why did it come" 等） | "what bird is this", "I looked it up" |
| `attachment` | 常連・愛着・戻ってくる関係 | "it comes back", "my favorite", "day 12" |
| `confusion` | 迷い・つまずき（**転用しないが設計に回す**） | "I didn't know how to", "where do I", "nothing happened" |

**(b) テーマタグ**（アプリ実体に対応。恣意的カテゴリを足さない）

`the-moment`（出会いの瞬間）/ `sound`（鳴き声・実録音）/ `calm-bgm`（癒し・環境音）/ `foodweb`（なぜ来たか・生態）/ `collection`（図鑑・会った日数）/ `onboarding`（最初の導線・つまずき）。

- テーマタグは `hook_bank.md` の4本柱（The Moment / Real Data / Sound Quiz / Ambience）と `aso_keywords.md` の意図軸（idle / cozy / relaxing sounds / collecting）に**そのまま橋渡しできる粒度**で設計している。
- `confusion` / `onboarding` は**マーケ転用しない**。企画部のUX改善インプット（開発部の「スクショ映え/導線」チケットの参考）へ回す。ネガティブ表現を宣伝に化粧しない誠実さ（原則4）。

### 2-4. 台帳の1行スキーマ（corpus.md の列）

`日付｜収集元(S1-S4)｜原文(英語・短句のみ)｜感情タグ｜テーマタグ｜転用可否(可/UX回し/保留)｜備考`

- **原文は「短い引用句」に限る**（一文〜数語）。段落丸ごと・個人が識別できる文脈は保存しない（§4プライバシー）。
- 収集元は種別記号のみ（S1〜S4）。**投稿者ハンドル・氏名・メール・DM相手の識別子は列に持たない**。

## 3. 転用（ASO文言候補 / 動画キャプション候補へ）

タグ付き表現のうち **転用可=可** のものだけを `derived_candidates.md` に落とす。手順は源→タグ→転用先の一本道。

### 3-1. (a) ASO文言候補への転用

**手順**:
1. `curiosity` / `calm` / `collection` タグの表現から、**検索者が実際に打ちそうな名詞句**を抽出（例: "relaxing bird sounds" と自発的に書いたテスター表現 → 語 `relaxing bird sounds game` の実需要の裏付け）。
2. `aso_keywords.md` §2 の採用語と**照合**し、(i)既存採用語を実ユーザー語が裏づける → 優先度の根拠として注記、(ii)採用語に無い自然な言い回しが繰り返し出る → **新規候補語として §1候補表へ追記提案**（採否はCEO実検索で判定・aso_keywords.md の運用に合流）。
3. `derived_candidates.md` に `出所行リンク / 抽出語 / aso_keywords.mdの対応語 or 新規提案` を記録。

**接続点**: `aso_keywords.md` §3「週次更新で回す」方針の**インプット源**になる。ただし文言化（タイトル/説明文への織り込み）はSNS部・実検索上書きはCEO（本書は候補の供給まで）。

### 3-2. (b) 動画キャプション候補への転用

**手順**:
1. `delight` / `surprise` / `the-moment` / `sound` タグの短句を優先抽出（ユーザーが出会いの瞬間を描いた生の一言は、7秒テストの冒頭字幕に最も強い）。
2. `hook_bank.md` の4本柱に割り当てる: `the-moment`→①The Moment / `sound`→③Sound Quiz / `calm-bgm`→④Ambience / `foodweb`→②Real Data Devlog。
3. **原文をそのまま字幕にはしない**（プライバシー＋トーン統一のため）。生表現を**種**として、SNS部が hook_bank の2行形式（Caption＋Shot）に**書き起こす**。本書は「この生声はこの柱のこの型に効く」という**割り当てメモ**まで。
4. 実録音を鳴らす提案には必ず `【要クレジット】` を継承（hook_bankの規約踏襲・原則4）。

**接続点**: `hook_bank.md` の各柱への**追加ドラフト種**を供給。文面量産はSNS部の担当（本書は踏み込まない）。

### 3-3. 転用フロー要約（一本道）

`源(S1-S4) → 週次タグ付け(SNS部) → corpus.md → 転用可を抽出 → derived_candidates.md → (a)aso_keywords.md §1候補へ / (b)hook_bank.md 各柱へ`
（企画部＝基準設計と原則監査、SNS部＝運用、CEO＝実検索上書き・返信・最終文責）

## 4. 原則ガード（このパイプライン自体が守ること）

| 原則 / 観点 | 本パイプラインでの守り方 |
|---|---|
| **原則1（受動的）** | 収集は週次・件数ノルマなし・0件を許容。転用時に「now / limited / hurry / don't miss」等の**時間圧・急かす文言に化粧しない**。`calm`/`the-moment`の静かな語のみをトーンに合わせて使う（§0ブランドトーン「押し売りしない」）。 |
| **原則2（罰しない）** | 撹乱・喪失の文脈の表現を「損をしたくないなら〜」の煽りに転用しない。 |
| **原則3（声と癒しは無料）** | ユーザーの絶賛を「有料でしか聴けない」示唆に接続しない。無料の癒しを主語にした表現のみ癒し訴求に使う。 |
| **原則4（生態に誠実）** | **生表現(corpus)と創作(derived)をファイルで分離。** ユーザーが誤った生態説明をしていても、それを事実として転用しない（GloBI/xeno-canto の裏付けがある範囲でのみ使用）。誇張・虚偽の生態説明を作らない。実録音言及は`【要クレジット】`継承。 |
| **原則5（かわいさ最優先）** | `delight`（かわいさ反応）を最優先ソースとして扱い、出会い・鳴き声・ドット絵への生の喜びを訴求の芯に据える。 |
| **プライバシー（§0-1・§2-4）** | 保存は**短い引用句のみ**。氏名・ハンドル・メール・DM識別子・特定できる文脈・スクショ実物は台帳に持たない。S4返信はCEO専任・自動生成禁止（§3-3・§5）。転用時に個人を推測させる固有情報を含めない。 |
