## 0. 前提(Phase S から不変・全部署常駐)
1. **予算$0。** 資源はCEOの時間(週最大6時間)+エージェント稼働のみ。有料広告・有償起用・有料ツール前提の提案は却下。
2. **主軸4本**=ASO / コミュニティ"参加" / founder-led短尺動画 / build in public。
3. **追う指標/無視する指標を固定。** 追う=事前登録(=現状はLP+Googleフォーム登録数)/活性化率(初回で新種初見+鳴き声5秒到達、目標30%+)/D1・D7/定性反応。無視=フォロワー・インプレ(報告可・意思決定不可)。
4. **英語ファースト。**
5. **7秒テストが門番。** ヒーロー動画は承認済み(`20260721_7s_test_v2.mp4`)。以後の短尺も7秒テストを満たす。
6. **トーン**=静か・誠実・押し売りしない。交渉不能5原則を全発信で厳守。

## 1. Phase 1 の目的とスコープ
準備(箱・素材)から**運用(実際に出す・参加する・測る・回す)**へ移る。Phase 1でやること:
- **発信の開始**(フックバンク→台本テンプレ→クロスポストで、少量・高品質に)。
- **コミュニティ"参加"**(規約マトリクス準拠。宣伝でなく貢献から)。
- **テスター獲得**(tester_channels準拠、クローズドテストへ)。
- **計測と週次実験**(サーバー基盤なし=Play Console+週次サーベイ+アプリ内ローカル統計)。
- **ストア掲載の準備継続**(公開はクローズドテスト完了後)。
- やらないこと: 大量投稿・複数チャネル同時全開・有料施策・比較/煽り。

## 2. 週次運営リズム(決定メモ§4を継承)
- **月曜**: CEOが各部長へ「今週の優先1件」を指示(下記メニューから)。
- **週内**: SNS部が投稿文面/参加文面を下書き → CEOが投稿・返信を実行(返信の自動化は厳禁)。
- **金曜**: 企画部が**週次1枚レポート**提出 → CEOが15分レビュー → 翌週判断。
- 全成果物は `marketing/phase-1/` に週別で保存。

## 3. 部署別 定常業務(実体3部+CEO)
### 企画部(頭脳)
- 週次1枚レポート(追う指標4つ+今週の実験と結果+来週提案1つ)。
- **実験の設計**(仮説→変更1つ→読む指標→判断基準)。
- 「ユーザーの言葉」タグ付け基準の運用監査(`user_language_pipeline.md`)。
- ASO実測の反映(コア5語の順位・競合をPlay検索で確認し文言更新指示)。
- 交渉不能原則の監査(全発信が原則に反しないか差し止め判断)。
### SNS部(対外文面)
- 週次の投稿文面: フックバンク→**台本テンプレ**→撮影用台本(CEO撮影)→**クロスポスト変換**で各面へ。
- コミュニティ参加の文面(規約マトリクスの"許可窓口/推奨フォーマット"に沿う)。
- コメント/DM/Redditの**返信下書き**(最終文責・投稿はCEO)。
- ストア掲載文言の磨き込み(公開はしない)。
### 開発部(コード・素材・計測)
- **ローカル統計画面**の実装(仕様書承認済み。活性化カウンタ+共有カード。壊さない検証必須)。
- 撮影素材の書き出し(スクショ・短尺)、スクショ映え改善チケット。
- **TelemetryDeck**の調査チケット(PII非収集・匿名設計・原則1両立可否)→ 導入判断はPhase 1ゲートで。
- 週次サーベイ/フォームの技術サポート。
### CEO(専任・委任禁止)
- アカウント開設・**投稿実行**・撮影(まとめ撮り可)。
- **コメント/DM/Reddit返信**(自動生成厳禁=ブランドの人格)。
- 7秒テスト等の合否判定。ハンドル確保・ストア入稿判断。

## 4. 投稿運用(何を・どこで・どれくらい)
- **チャネル優先度(確定)**: **①TikTok/Reels/Shorts(同一縦動画を1本化=第一チャネル)②X(build in public・週1定常=第二)③Reddit/Discord(参加=信頼構築、宣伝は規約許可時のみ)。**
  - **なぜTikTok系が第一か**: (1)Phase 1で追う指標(事前登録・活性化率・D1/D7)は**エンドユーザー**指標で、それを動かすのはcozy/自然/casual消費者=短尺ネイティブ層。Xの build in public は開発者/AI/インディー層に届き複利だが、消費者アプリの登録・活性化には直結しにくい。(2)潤沢な素材(7秒テスト・実写ストーリー・フックバンク主力=型1 Reveal/型4 Sound Quiz/型2 Ambience)が短尺ネイティブ。(3)TikTok/Reelsは**フォロワーゼロでもアルゴリズム発見**が効く=$0・無名スタートに最適。(4)クロスポストSOPで**1制作=3面**。
  - **Xの役割(第二・週1)**: 捨てない。「複利の物語ログ + **テスター獲得の導線**(インディー/Android開発者は優秀なテスター)+ 将来のプレス」。Xの主産出=フォロワー/インプレは§0で"判断に使わない"と固定済みなので第一にはしない。
  - **併走はしない**: 週6時間ソロで2面全開は品質・トーンが崩れる。第一を1つに絞り、Xは低頻度定常に固定。
- **週の内容ミックス(例・少量高品質)**: 短尺動画1本(フックバンク4本柱を順に)/ Xのbuild in public 1〜2投稿(開発の意思決定・データの発見・スクショ)/ コミュニティ参加(貢献)数件。
- **1本使い回し**: 縦動画1本を作り、クロスポストSOPでキャプション/タグだけ各面に変換。
- **birding系(r/birding等)は宣伝でなく信頼構築の場**(原則4)。cozy/incremental/IndieDev系を主戦場。

## 5. 実験サイクル(test → measure → decide)(確定)
- **実験=1週に1つ・1変数**を仮説付きで動かす。**同時に2変数を動かさない。**
- **判定は"硬い信号+定性"で行い、再生数では判定しない**(再生数は§0で判断に使わないと固定)。
  - 硬い信号=(a)登録フォーム転換 (b)テスター応募数 (c)活性化率(ローカル統計リリース後)。ソフト=コメント/DM/週次サーベイの言葉。
- **変数はラダー化(第一チャネル上)**: 例 第1週=型1 vs 型4 の反応 → 第2週=字幕1行目の言い回し → 第3週=投稿曜日/時間 → 第4週=末尾CTA→登録タップ → 第5週=テスター募集文面→応募転換 → 第6週=型2 Ambience vs 型1 Reveal。
- **実験は第一チャネルに1本のみ載せる**。X(build in public)は"実験"ではなく複利ログとして別トラックで淡々と(チャネル間で変数を混ぜると交絡する)。
- **判断**: 金曜レポートで「効いた/効かない」を言語化し、翌週に1つだけ動かす。

## 6. 計測(決定メモ§決定2 準拠・サーバー基盤なし)
- (a) **Play Console標準統計**を企画部が週次転記。
- (b) **週次マイクロサーベイ**(`weekly_survey.md`、CEO配信)。
- (c) **アプリ内ローカル統計画面**(端末内のみ・ユーザー操作で書き出し)。**活性化=初回で新種初見かつ鳴き声5秒**をローカルカウンタで記録。
- 外部基盤(TelemetryDeck)は本Phaseで**調査のみ**、導入は次ゲート判断。

## 7. テスター獲得
- `tester_channels.md`準拠。既存LP登録/知人テスターを最優先母体に、Android/ベータ系で$0募集。募集文に「Android限定/鳴き声は無料/デイリー義務なし」を必須要素。

## 8. ストア掲載
- 文言・スクショ構成は準備済み(`store_listing_prep.md`)。**公開作業はクローズドテスト完了後**(決定3-3)。入稿判断はCEO。

## 9. 禁止事項(不変)
