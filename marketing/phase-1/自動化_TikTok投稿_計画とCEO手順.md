# TikTok投稿の自動化 — 計画とCEO登録手順（Phase 1 / 2026-07-23確定）

**決定**: 投稿の自動化は**TikTokのみ・$0**で進める。**Xは当面手動**（2026年2月にX無料枠廃止＝リンク入り$0.20/本のため。詳細は memory `reference_social_posting_api_2026`）。
**担当分担**: 外部API仕様の実測=PJ管理 / 実装=開発部(dev) / **開発者アプリ登録・OAuth認可・投稿の最終公開タップ=CEO**（人格・アカウント主体のため代行不可）。

---

## 0. いま正直に知っておくこと（期待値調整）
TikTok Content Posting API には2つの経路があり、**できることが違う**:

| 経路 | scope | 審査 | 自動化できる範囲 |
|---|---|---|---|
| **下書き転送 (upload to inbox)** | `video.upload` | **不要・$0** | 完成動画を**端末のTikTok下書きに自動投下**。**キャプション入力・公開はアプリ内で人が実施** |
| **一発公開 (direct post)** | `video.publish` | **要・2〜6週間** | **キャプションごとAPIで公開**（真の一発自動化）。審査前は投稿がSELF_ONLY(非公開)になる |

→ **審査前でも"下書き転送"は今日から$0で使える**が、実利は「ファイル転送の自動化」まで。
→ **一発公開は審査通過が条件**。審査は数週かかるので**今から申請の時計を回す**のが得。

## 1. 全体の段取り
1. **CEO**: TikTok開発者アプリを登録（下記§2）→ Client Key / Client Secret を取得し、開発部に安全に共有。
2. **開発部**: 連携スクリプトを実装（OAuth認可→トークン保存/更新→動画を下書き投下）。壊さない検証は本体コードに触れないため対象外だが、秘密情報は**リポジトリに絶対コミットしない**（`.env`＋`.gitignore`）。
3. **審査前運用**: `python post_tiktok.py 動画.mp4` 一発で、動画がCEOの端末TikTok下書きに出現 → CEOがキャプション（私が用意）を貼って公開。
4. **審査申請**: 上記が動いたら direct-post(`video.publish`)の審査を申請 → 通過後は**キャプションごと一発公開**に昇格。

## 2. CEO登録手順（TikTok開発者アプリ・約15〜20分）
> ブラウザで https://developers.tiktok.com にログイン（@toriscollection.rk と同じTikTokアカウントで）。

1. **開発者登録**: 右上から Developer 登録（無料）。組織名は個人名/ブランド名でOK。
2. **アプリ作成**: 「Manage apps」→「Connect an app」→ アプリ名 `Toris Collection Poster` 等。
3. **プロダクト追加**:
   - **Login Kit** を追加（OAuth認可に必須）。
   - **Content Posting API** を追加。**scope は `video.upload`**（まずは審査不要のこれ）。※`video.publish` は後日、審査申請時に追加。
4. **Redirect URI 設定**: `http://localhost:8080/callback` を登録（ローカルでOAuthを受けるため。開発部が使う）。
5. **キー取得**: アプリの Basic information から **Client key** と **Client secret** を控える。
6. **開発部へ共有**: Client key / Client secret を**安全に**私(PJ管理)経由で開発部へ。※これらは秘密。**スクショをそのままSNSや公開場所に貼らない**こと。
7. （後日）**審査申請**: スクリプトが動いたら、アプリ設定から Content Posting API の**本番審査**を申請（`video.publish`＝一発公開の解放。2〜6週間）。

## 3. 未確認（開発部が実装時に実測）

- $0を守る（TikTok API利用は無料。Xは有料化したので手動維持）。
