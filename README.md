# Toris Collection

**会いに行った鳥たちが、あなたのラジオで毎日鳴く——生態学で動く"生きた庭"のアンビエント・コレクション**

植物を植えて鳥を呼び、近くまで会いに行く。出会った鳥はあなたの「ラジオ」に加わり、季節や天気、群れや掛け合いとともに毎日違う声で鳴く。背後では本物の食物網・共起ネットワーク・撹乱（嵐や伐採）といった生態学モデルが静かに動き続けている。

**公開URL**: https://toriscollection-test202605.streamlit.app/

## 1. プロダクトの背骨（いちばん大事な1つ）

> **ラジオがコア（毎日帰る場所）。儀式（鳥に会いに行く）は、ラジオに出す鳥を増やす手段。**

```
会いに行く（儀式） → 出会った鳥がラジオに加わる → ラジオが豊かになる
                  → もっと違う顔ぶれを聴きたくなる → また会いに行く
```

このループを回し続けることが、すべての設計判断の上位にあります。新しい機能は「**ラジオを目に見えて豊かにするか／ユーザーを引き戻すか**」のどちらかを満たさなければ入れません。

- **ラジオ＝蓄積するコレクション**：一度会った鳥は記録（observed/discovered）に残り、減りません。嵐や伐採で庭の植物が倒れて鳥が庭を去っても、ラジオでは今までどおり鳴きます。
- **儀式＝出会いの入口**：近くまで「会いに行く」瞬間。捕まえる仕組みはありません。
- **生態学は魔法の裏付け**：「誰と誰が一緒に鳴くか」は共起ネットワークが決め、ナレーションで言語化されます。

## 2. プレイヤーの1日（コア体験）

1. **土地（バイオーム）を選ぶ**：京都・シドニー・シャーロットの3つから。
2. **植物を植える**：土地に合う植物を選ぶ。植物→昆虫→鳥の食物網が、来る鳥を決める。
3. **不在中に世界が動く**：アプリを閉じている間も、経過時間に応じて生態系が進化（鳥の到着・退去、嵐・遷移、落とし物）。
4. **戻ると"今日"がある**：留守中に来た新顔はラジオで🌟点灯。さらに「今日の庭」（1日1回・全員共通の一羽）が毎日待っている。
5. **ラジオを聴く**：出会った鳥が季節・時間帯・群れ・掛け合いとともに鳴く。
6. **会いに行く（儀式）**：まだ近づいていない鳥に距離を詰めて出会い、ラジオに加える。
7. **落とし物を集める／みんなの庭を覗く**：羽根や小枝の収集、みんなの観察が集まった匿名の「生きた地図」。

## 4. アーキテクチャ（3つの層）

コードは役割で3層に分かれています。**この分離が、将来のスマホアプリ／バックエンド移行のための一番大事な投資**です（詳細は `ARCHITECTURE_PLAN_2026-06.md`）。

```
┌─────────────────────────────────────────────────────────┐
│  プレゼンテーション層（差し替え可能・使い捨ての"皮"）          │
│    app.py        … Streamlit の画面・10タブ制御              │
│    radio.py      … ラジオUI＋Web Audio（JS生成）             │
│    ritual.py     … 儀式UI（距離メカニクス・スプライト）       │
│    community.py / daily.py … 各タブの描画                   │
└───────────────┬─────────────────────────────────────────┘
                │ 呼ぶ
┌───────────────▼─────────────────────────────────────────┐
│  コア層（純粋ロジック・I/Oなし・テスト可能・移植可能）          │
│    engine.py       … 食物網・鳥の出現確率・1ターン進行        │
│    ecology.py      … 共起ネットワーク（誰と誰が一緒に鳴くか）  │
│    disturbance.py  … 撹乱と遷移（嵐・落雷・伐採→新しい鳥）     │
│    flock.py        … 群れサイズ（よく会うほど厚い声）          │
│    daily.py        … 今日の庭（日付シードの一羽）             │
│    absence_loop.py … 不在中の時間進化（上記を束ねる）         │
│    community.aggregate_atlas … 匿名・集合の集計             │
│    mementos.py     … 落とし物の抽選                        │
└───────────────┬─────────────────────────────────────────┘
                │ 読む/書く
┌───────────────▼─────────────────────────────────────────┐
│  データ／IO層（外部とのやり取り・将来DBに差し替え予定）         │
│    sheets_client.py     … Google Sheets 読み書き           │
│    species_loader.py    … 種データのロード窓口（Sheets/シード）│
│    xc_client.py         … xeno-canto 鳴き声取得・キャッシュ   │
│    freesound_client.py  … 環境音取得・キャッシュ             │
│    observation_log.py   … 観察記録ラッパー                  │
│    data.py              … シードデータ（35種ずつ）            │
└─────────────────────────────────────────────────────────┘
```

**コア層はどこにも依存しない純粋計算**なので、そのままユニットテストでき、将来 Flutter/React Native のスマホアプリ＋Python API に移すときも**書き直さずに再利用**できます。

## 6. 主要な生態モデル（短く）

- **鳥の出現確率** = 気温適合度 × バイオーム補正 × 食物係数 × レア度係数 × 0.5（`engine.py`）。レア度は GloBI 由来の PageRank 中心性で補正。
- **共起（誰と一緒に鳴くか）** = 気候ニッチ重なり × ギルド係数 × 競争抑制（`ecology.py`）。「同じ餌＝仲間」は誤り（競争排除則）として補正済み。
- **撹乱と遷移** = 嵐/落雷/伐採が低頻度で植物を倒し、跡地に別の植物が芽吹き、別の鳥が来る（`disturbance.py`）。罰ではなく「庭が移ろう」体験。
- **群れ** = よく会った社会性の鳥ほど複数で鳴き、声が厚くなる（`flock.py`）。
- **不在中ループ** = 経過時間（5分→24h+）を 0〜6 サイクルに変換し、上記をまとめて進める（`absence_loop.py`）。

### 種数（現状）

| バイオーム | 植物 | 昆虫 | 鳥 |
|---|---|---|---|
| 京都 | 13 | 9 | 14 |
| シドニー | 11 | 6 | 10 |
| シャーロット | 11 | 8 | 11 |
| **合計** | **35** | **23** | **35** |

## 7. データの保存先（Google Sheets スキーマ）

各テスターのデータは `tester_id` で1つのスプレッドシートに同居（マルチテナント）。

| シート | 内容 |
|---|---|
| `testers` | テスターIDと表示名 |
| `field_state` | 現在のバイオーム・滞在中の鳥・最終アクセス時刻 |
| `plantings` | 植えた植物の履歴（active/removed） |
| `bird_visits` | 鳥の訪問記録（滞在中・不在中）と理由 |
| `collection` | 図鑑（初回・最終観測・累計回数） |
| `mementos` | 落とし物の獲得履歴 |
| `bird_notes` | 図鑑メモ（互換データ） |
| `access_logs` | 行動ログ（あしあとの素材） |
| `species_birds` / `species_plants` / `species_insects` | 種データ（任意。あれば Sheets が優先、なければ `data.py` のシード） |

> **種の追加は Excel/Sheets で可能**：`species_*` シートにヘッダー付きで1行1種を足すと、次回起動で反映されます（列仕様は `toris_collection/data_types.py`）。植物の `disturbance_sensitivity`/`successional_role`、鳥の `flock_max` などの任意列も対応。

## 8. ローカルでの起動

```bash
cd toris_collection
python -m pip install -r requirements.txt

# Google Sheets を使う場合は credentials.json をプロジェクト直下に配置
#（無くてもシードデータで動作。Sheets 連携部分はスキップされます）

python -m streamlit run app.py
# → http://localhost:8501
```

任意の API キー（あると鳴き声・環境音が有効化）:

- `xc_api_key.txt` … xeno-canto（鳴き声）
- `freesound_api_key.txt` … Freesound（環境音）

## 9. テスト

純粋ロジックは pytest 不要・標準ライブラリのみで回せます。

```bash
cd toris_collection
python tests/test_disturbance.py   # 撹乱・遷移      9ケース
python tests/test_flock.py         # 群れ           8ケース
python tests/test_daily.py         # 今日の庭        7ケース
python tests/test_community.py     # 集合アトラス     7ケース
```

コア層が I/O に依存しないため、重い依存（gspread 等）を引かずに高速に通ります。

## 10. デプロイ（Streamlit Community Cloud）

1. GitHub に push（`.gitignore` で `credentials.json` 等は除外）。
2. https://share.streamlit.io/ で「New app」→ リポジトリ・ブランチ・main file path（`toris_collection/app.py`）。
3. Advanced settings → Secrets に TOML で認証情報を貼り付け（`[gcp_service_account]` ほか）。
4. Deploy。push のたびに自動再デプロイ。

## 11. これからの方向（ロードマップ要約）

背骨ループは「会う→ラジオが豊かに→戻る」の**両方向が実装済み**。次は機能追加ではなく、

- **① テスターでの検証**：既存ログ（access_logs/あしあと/collection）で「戻ってくるか・豊かに感じるか」を測る。
- **② 基盤整備**：Google Sheets → 本格DB への移行を見据え、コア層を移植可能に保つ。これが Google Play 配信・スケール・本格ソーシャルの共通ゲート。詳細は **`ARCHITECTURE_PLAN_2026-06.md`**。

「地域エクスパンション・季節パス・寄付連動・相棒」などは、①が「ループが回る」と示してから。
