# 🐦 #Toris Collection#

土地を選び、植物を植え、時間が経つのを待つ。やってきた鳥たちの声に耳を澄ますコレクションアプリ。

## 3つの柱

- **コレクション**: 鳥を集める楽しみ(図鑑は退去しても残る)
- **癒し**: 滞在中の鳥たちの鳴き声を聴く
- **グリーン**: 植物→昆虫→鳥の食物網を組み立てて学ぶ

## コアメカニクス

プレイヤーが直接鳥を「捕まえる」のではなく、**環境を整えて来てもらう**設計。

```
土地(バイオーム)を選ぶ
 → 植物を植える・水場を設置する
 → 時間経過で昆虫が発生
 → 食物網がつながった鳥が確率的に飛来
 → 環境が合わない鳥は退去(図鑑からは消えない)
```

## 設計上の重要な判断

**1. ネットワークは種ペアの直接エッジに統一**
植物・環境・昆虫・鳥の4種類のノードのみ。資源ノードは持たない。GloBI互換。

**2. 渡り鳥タグを持たない**
季節性は気温適合域だけで自動的に表現される。
オオルリ(12-22℃): 冬 p=0.02 / 夏 p=0.26。
→ 世界展開時に地域ごとの「渡り」定義が不要。

**3. 出現確率 = 気温適合度 × バイオーム補正 × 食物網スコア × レア度係数**
食物網スコアは NetworkX で鳥への流入エッジ重み総和を飽和関数で正規化。

**4. 力学モデルによる放射状可視化**
spring_layout でレイアウト。植えた植物(入力)と来た鳥(出力)は濃色・大、
昆虫・未訪問鳥は淡色・小で、一目で何が起きているか読める配色。

**5. GloBI連携で世界展開可能**
シードから始めるが、学名があれば GloBI API から実データを取得・検証できる。

## Small Start 構成

- バイオーム: 3種類(温帯林・里山・照葉樹林 = 北海道・京都・沖縄タイプ)
- 植物: 14種 / 環境: 1種(ため池)
- 昆虫・中間動物: 8種
- 鳥: 20種
- 1ターン = 半月

## 起動

```bash
pip install -r requirements.txt
streamlit run app.py
```

## GloBI キャッシュの事前ビルド(推奨)

初回に一度実行すると、全鳥の相互作用データをローカルにキャッシュします。
UI上の「🌐 GloBI」タブでの検索が高速になります。

```bash
python build_globi_cache.py
```

約20秒で完了(鳥20種 × 0.5秒ウェイト)。

## ファイル構成

```
data.py              種と相互作用のシードデータ
engine.py            ネットワーク構築・確率計算・レイアウト
app.py               Streamlit UI(6タブ構成)
globi_client.py      GloBI API クライアント + キャッシュ
build_globi_cache.py 初回キャッシュビルダー
requirements.txt     依存関係
```

## 今後の拡張

**Phase A (完了)**: GloBI API連携。学名から実際の食物網を取得して検証。

**Phase B**: 日本以外の地域・種に拡張。
- xeno-canto API から鳴き声音源を取得(CCライセンス)
- Open-Meteo 等の気候API連携で実際の気温を反映
- GloBI の bbox 指定で地域ごとの食物網を自動構築

**Phase C**: Streamlit → FastAPI + Next.js へ移行。
位置情報連動(プレイヤーの居住地の気候が反映される)。
