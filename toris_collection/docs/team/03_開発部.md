# 開発部向け — 方針とルール

対象: 実装・データ・インフラ・品質を担う人。

## 技術スタック(現状)
- **フロント/実行**: Streamlit(`toris_collection/app.py` がメイン)。ラジオは iframe 内 Web Audio。
- **永続化**: **セーブコード方式(ローカル保存)が主経路**(`save_code.py`、2026-07-04・CEO承認)。
  個人の進行データはブラウザの session state に留め、`st.download_button`/貼り付けで
  書き出し・復元する(サーバーには送らない)。ログイン概念は廃止し、`render_login_screen()`
  は「新規スタート/セーブコード読込」の2択。Google Sheets(`sheets_client.py`)は
  ベストエフォート(失敗しても無視)で残置。将来の「みんなの庭」復活時のみ再利用想定。
  `st.secrets`→env→file の順で認証・キー解決する仕組み自体は変更なし。
- **生態エンジン**: `engine.py`(plant→insect→bird の有向グラフ+到来確率)、
  `disturbance.py`(撹乱=純減)、`absence_loop.py`(不在中の時間進行)。
- **データ**: `data.py`(シード)/ `species_loader.py`(Sheets優先→シード)。GloBI=`globi_client.py`。
- **音声**: `xc_client.py`(xeno-canto)/ `freesound_client.py`(環境音)/ `radio.py`。
- **デプロイ**: Streamlit Cloud(branch=main、main file=`toris_collection/app.py`)。

## 開発のルール(守ること)
- **今のものをベースに**。作り直しでなく既存の延長で足す(ユーザー方針)。
- **壊さない検証を必須に**: 変更後は毎回
  - `python -c "import ast; ast.parse(...)"`(構文)
  - Streamlit `AppTest` で起動時例外0
  - `tests/test_*.py` 全 pass
  - ラジオJSは Node `--check` + Playwright(既存 Chromium)で実挙動確認
- **上記だけでは不十分(2026-07-11・CEO指摘)**: 「エラーが出ない」ことしか保証できず、
  「意図通りに動くか」を見ていない。仕様・体験・バグ修正に関わる変更は、以下の2段階を追加で必須とする。
  1. **着手前に受け入れ基準を文書化する**: 「直った/追加された状態が何をどう満たせば正しいか」を
     1〜3行で先に書く(バグなら「再現手順」+「直った後どう見えるべきか」のセット)。
     企画部(kikaku)の既存仕様(`CONCEPT_v2.md`/`01_企画部.md`等)に既にある場合はそこを引用するだけでよい。
     受け入れ基準が無いまま「とりあえず直す」実装はしない。
  2. **実装後、その受け入れ基準に対して検証する**: 上記の技術的検証(構文・例外0・tests・Playwright)は
     最低ラインとして維持しつつ、受け入れ基準の各項目を実際に操作(またはAppTest/Playwrightでの再現)して
     満たしているかをYes/Noで確認し、報告に含める。数値バランス・体験に関わる変更は企画部が
     受け入れ基準との一致を確認してから「完了」とする(実装が終わっただけでは「完了」と呼ばない)。
     デバッグログ追加や原因調査だけの変更は「対応中」であり、受け入れ基準を満たす修正が
     入るまでは完了報告しない。
- **秘密情報をコミットしない**(`.gitignore`: credentials.json / xc_api_key.txt / secrets.toml)。
  キーは Streamlit secrets / 環境変数から読む(実装済の解決順を踏襲)。
- **Sheets 障害でアプリを止めない**: 書き戻しは `_sheets_safe`、読みは try/except → 既定値。
- **恣意ロジックを足さない**: 種の関係は GloBI 由来。動物連鎖も eats/eatenBy/preysOn の向きで表現。
- **ドット絵**: 新種は `SPRITE_ALIASES` で既存流用可(後追いで専用絵に差し替え)。
- **HTML埋め込みは `components.html()` を使う**(`import streamlit.components.v1 as components`
  でモジュールとして import してから呼ぶこと)。`st.iframe()` という同名APIは存在しない
  (2026-07-04、ラジオ・儀式・ネットワーク図で誤用していたバグを修正済み)。
  `components.v1.iframe()` はURL埋め込み専用で別物なので混同しないこと。

## 未着手/配線待ちの差込点
- **シミュ・ラボ**: `engine`/`absence_loop` を「制約なし・早送り」で回す別タブ。
  ラボの結果は session だけで完結し、**本編の bird_days・図鑑・Sheets に書かない**。
- **おやつ=餌台の本編配線**: `feeder_chain.resolve()` を run_turn 前に呼び、
  `wary_arrival_multiplier` を到来確率へ乗算。リス/Hawk は `role` でラジオ/図鑑から除外。
- **商用モード**: 監査後に secrets `commercial_only=true`。Google Play収益化(広告/Play Billing)
  に着手するタイミングで判断(`ROADMAP_GOOGLE_PLAY.md` 参照)。
- **PWA化(TWA準備)**: Web app manifest + 最小サービスワーカーの追加。バックエンド独立
  (Firestore移行等)は不要と判断(ローカル保存MVPによりSheetsのスケール問題が個人データ側
  では解消したため)。詳細は `ROADMAP_GOOGLE_PLAY.md`(2026-07-05改訂)を参照。

## 環境メモ
- 開発サンドボックスから **GloBI は 403**(組織ポリシー)。GloBI 実データ取得は Cloud で。
- Playwright は既存 Chromium を `executable_path` 指定で使う(再ダウンロード不要)。
