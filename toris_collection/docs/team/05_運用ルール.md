# 05_運用ルール — 承認ライン・報告フォーマット・エスカレーション

最終更新: 2026-07-03
位置づけ: `00_共通サマリ.md`(交渉不能の原則)・`01_企画部.md`・`02_SNS部.md`・`03_開発部.md`・
`04_社長.md` の4部門ドキュメントには、部署間の日々のワークフロー(承認ライン・報告様式・
エスカレーション手順)がなかったため、それを補う運用文書として追加。

## 1. 「大きく変える」の具体基準(事前承認が必要なライン)

`00_共通サマリ.md` の「交渉不能の原則」を踏まえ、「コード・データ構造・仕様を大きく変える作業」を
部署ごとに具体化する。

### コード・データ構造(開発部)
- Google Sheets のシート構成・列構成の変更、DB/バックエンドの移行(Storage層導入等)
- `toris_collection/HANDOFF_2026-06.md` §8 の「確定済みの設計判断」を覆す変更
- 背骨(ラジオ=コア、`HANDOFF_2026-06.md` §1-1)に触れる変更
- 本番デプロイ、Google Play 提出などユーザーに見える形で公開される操作
- 既存タブ構成・コアループの大幅な作り直し

### 仕様(企画部)
- `toris_collection/CONCEPT_v2.md` の「機能の再定義」表(★★★の核機能)を変更する提案
- 収益モデル・課金に関わる新規提案
- 「やらない/後回し」に確定済みの項目(天気連動・ガチャ等)を覆す提案

### 発信(SNS部)
- 実際の投稿・提出そのもの(ドラフト作成は事前承認不要、送信・提出はCEOが行う)
- 教育コラボ等、外部組織との関係構築を伴う発信

### 事前承認が不要なもの(通常業務として進めてよい)
- バグ修正、既存テストの追加、軽微なUI調整
- 既存方針の範囲内での種・データ追加(生態学的に妥当なもの)
- 文書の整合修正・誤字修正
- 文面ドラフトの作成(投稿はしない)

判断に迷う場合は「小さく見えても、①交渉不能の原則に触れる可能性がある、または
②ユーザーに見える形で公開される、のどちらかに該当すれば事前承認を取る」を原則とする。

## 2. PJ管理部長の振り分けフロー

1. CEOの指示を受けたら、まずどの部署の担当領域(`01_企画部.md`/`02_SNS部.md`/`03_開発部.md`)かを判断する。
2. 複数部署にまたがる場合は、それぞれにサブエージェントを起動し、必要な範囲の指示を渡す。
   (背景説明はサブエージェント側は文脈を持たないため、必要な情報を指示に含める。)
3. §1の基準に照らして事前承認が必要な内容が含まれる場合は、実行前に計画をCEOに提示する。
4. 各部署の成果物を受け取ったら、部長がまとめてCEOに日本語で報告する。

## 3. 報告フォーマット(CEOへの報告)

日本語で、以下の構成を基本とする(簡潔で良い、形式的な見出しの羅列は不要):

- **何をしたか**: 担当部署ごとの実施内容を簡潔に
- **結果**: 開発部の場合は「壊さない検証」(`03_開発部.md` 参照)の結果を必ず含める。
  仕様・体験・バグ修正に関わる変更は、`03_開発部.md`の「受け入れ基準→検証」2段階
  (着手前に企画部の仕様と合わせた受け入れ基準を文書化し、実装後にそれを満たすか確認したか)
  も併せて報告する(2026-07-11追加)。
- **承認が必要な事項**: あれば明示し、実行前であることを明確にする
- **次のアクション**: 次に何をするか、CEOの判断待ちの点があれば明示

## 4. エスカレーション

- 交渉不能の原則(`00_共通サマリ.md`)に抵触する提案・依頼を検知した場合、
  どの部署が発案・受領したものであっても、実行前に部長がCEOに報告し判断を仰ぐ。
- 部署間で方針が矛盾する場合(例: 企画部の提案が開発部の「確定済み設計判断」と衝突する)、
  部長が矛盾点を整理してCEOに判断を仰ぐ。部署間の押し切りで処理しない。

## 5. ドキュメントの更新ルール

- `00_共通サマリ.md`(全社共通の交渉不能の原則)の変更は必ずCEO承認を要する。
- `01_企画部.md`/`02_SNS部.md`/`03_開発部.md` は、該当部署の業務範囲内であれば部長経由で
  更新提案できるが、`00_共通サマリ.md` の原則との整合を常に確認する。
- `04_社長.md`(経営判断ポイント)の変更はCEO(社長)自身の領域であり、部長が代筆・変更提案する
  場合も必ずCEO確認を経る。
- `05_運用ルール.md`(この文書)の変更もCEO承認を要する(運用の根幹のため)。
