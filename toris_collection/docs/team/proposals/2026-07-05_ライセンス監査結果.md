# ライセンス監査結果(2026-07-05・開発部)

Google Play ロードマップ Phase 0 対応。`license_audit.py`
(`species_license_summary`/`audit`)と `check_xeno_canto_coverage.py` を使った
xeno-canto録音ライセンス監査の実行結果。

## 結論(先に要点)

**この開発サンドボックスには xeno-canto APIキーが無く、実際の監査(全47種の
ライセンス内訳集計)を実行できなかった。** 集計結果の代わりに、実行環境の
確認結果と、キーを用意すれば即実行できる手順を報告する。

## 確認したこと

1. **APIキーの有無**: `xc_client._load_api_key()` の解決順(Streamlit secrets →
   環境変数 `XC_API_KEY` → `xc_api_key.txt`)を1つずつ確認したが、いずれも
   未設定だった。
   - `xc_api_key.txt`: 存在しない
   - 環境変数 `XC_API_KEY`: 未設定
   - `.streamlit/secrets.toml`: 存在しない
   - `xc_client.is_enabled()` → `False`(実行して確認済み)

2. **xeno-canto自体への到達性(GloBIの403とは別問題であることの確認)**:
   キー無しで `https://xeno-canto.org/api/3/recordings` に直接アクセスしたところ、
   **HTTP 401**(`"Missing or invalid 'key' parameter"`)が返ってきた。
   これは「サービス自体には到達できているが、認証が必要」という正常な応答であり、
   `globi_client.py` が受ける組織ポリシーによる **403** ブロックとは別種の応答。
   つまり **xeno-cantoはこのサンドボックスからブロックされていない**。
   APIキーさえ手に入れば、この環境でも `license_audit.audit()` を実際に
   実行できる見込みが高い。

3. **対象種数**: `species_loader.BIRDS` は現在 **47種**(シードデータ使用、
   Sheets接続なしのためシードにフォールバック)。全種に `scientific`
   (学名)が設定済みで、監査対象から漏れる種はない。

## キーの入手方法(`xc_client.py` 冒頭コメント準拠)

1. https://xeno-canto.org/account/register で登録(無料)
2. https://xeno-canto.org/account/api-key でAPIキーを取得
3. 次のいずれかで設定:
   - ローカル開発: `toris_collection/xc_api_key.txt` にキーだけ1行で保存
     (`.gitignore` 済みなのでコミットされない)
   - 環境変数: `XC_API_KEY=<キー>`
   - Streamlit Cloud: secrets に `xc_api_key = "<キー>"`
4. キー設置後、`cd toris_collection && python check_xeno_canto_coverage.py`
   で音源カバレッジ、または以下でライセンス内訳を集計できる:

   ```python
   from species_loader import BIRDS
   from license_audit import audit, summarize
   rows = audit(BIRDS)
   print(summarize(rows))
   ```

## 2026-07-08 追記: 実監査結果

CEOが xeno-canto APIキーを取得し `toris_collection/xc_api_key.txt` に設置。
これを用いて実監査を実行した。**47種全種を最後まで監査でき、レート制限等で
中断した種はゼロ**(APIキー末尾4桁 `...2d92` として読み込み確認済み。値自体は
本ドキュメントにも記録していない)。

### 実行したこと

1. `xc_api_key.txt` の存在確認(内容は伏せ、文字数=40のみ確認)。
2. `python check_xeno_canto_coverage.py` を実行(音源カバレッジ)。
   - Windowsコンソールの既定エンコード(cp932)では絵文字が出力できず
     `UnicodeEncodeError` になったため、`PYTHONIOENCODING=utf-8` を付けて
     再実行した。**これはコード側のバグではなく実行環境の文字コード設定の
     問題であり、`check_xeno_canto_coverage.py` 自体は変更していない。**
3. 以下のコードで `license_audit.audit()` / `summarize()` を実行:
   ```python
   from species_loader import BIRDS
   from license_audit import audit, summarize
   rows = audit(BIRDS)
   print(summarize(rows))
   ```
4. 監査ツール(`license_audit.py` / `check_xeno_canto_coverage.py`)・
   種データ(`data.py` / `species_loader.py`)・`xc_client.COMMERCIAL_ONLY`
   は一切変更していない。実行のみ。

### 商用化にあたって影響が大きい種

すべて `data.py` の種定義(`rarity` 値、小さいほど頻繁に来る)を確認した上での所見:

- **シジュウカラ**(*Parus minor*、rarity 0.3=よく来る・京都バイオームの代表種の1つ)
  — 録音221件が**全てNC**(商用可0件)。京都で最初に会う鳥の一つとして想定される
  頻出種が商用可0件なのは影響が大きい。
- **ウグイス**(*Horornis diphone*)— 日本の春の代表的な鳴き声。53件全てNC。
  かわいさ・癒しの核となる声の一つだけに、商用モードで無音化する影響が大きい。
- **ワライカワセミ**(Laughing Kookaburra、*Dacelo novaeguineae*)・
  **キバタン**(Sulphur-crested Cockatoo)・
  **モモイロインコ**(Galah)— シドニーバイオームを象徴する人気種3種が
  揃ってNC-onlyだった。シドニー体験全体の"目玉"級の鳴き声が失われる。
- 一方、京都の**スズメ・メジロ・ツバメ・カワセミ**、シドニーの
  **ゴシキセイガイインコ・カササギフエガラス**、シャーロットの多くの種は
  商用可録音があり、各バイオームの入口となる定番種は概ねカバーされている。
- 完全に音源のないコゲラは、既に把握済みの1種のみで、今回の監査で変化なし。

### 次のアクション(CEO判断待ち)

`commercial_only` を有効化する/しないの判断材料として:

- **有効化した場合**: 47種中27種(57.4%)は鳴き声を維持できるが、
  シジュウカラ・ウグイス・ワライカワセミなど「顔」となる人気種19種
  (40.4%)は無音になる(録音自体は今もNCとして存在するが使えなくなる)。
  無音時のUI・図鑑表示(「録音なし」扱いにするか等)は企画部と要調整。
- **有効化しない場合**: 広告(商用)導入の前提(`CONCEPT_v2.md` L73)を
  満たさないため、広告つき収益化はこのままでは着手できない。
- 折衷案の検討余地(企画部・社長判断領域、開発部からの提案ではない):
  a) NC-only種は鳴き声UIを非表示にせず「録音準備中」の位置づけで残す、
  b) NC-only種のうち特に人気の高い種(シジュウカラ・ウグイス等)だけ
  他の合法な音源(自主録音・別ライセンス音源など)を別途調達する、
  c) 広告を「非商用の範囲で許容されるモデル」(例: 個人開発者の非商用扱いが
  可能な広告ネットワークがあるか)で検討する。
  いずれも**交渉不能の原則3「鳥の声と癒しは常に無料」には抵触しない**
  (無料/有料の話ではなく、ライセンス上使える録音の範囲の話)。
  ただし実施判断・切替そのものはCEO/社長マター。開発部はコード変更を伴う
  対応(a〜c)の実行前に必ず計画を提示し承認を得る。
