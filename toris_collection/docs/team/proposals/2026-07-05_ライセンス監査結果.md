# ライセンス監査結果(2026-07-05・開発部)

Google Play ロードマップ Phase 0 対応。`license_audit.py`
(`species_license_summary`/`audit`)と `check_xeno_canto_coverage.py` を使った
xeno-canto録音ライセンス監査の実行結果。

## 結論(先に要点)

**この開発サンドボックスには xeno-canto APIキーが無く、実際の監査(全47種の
ライセンス内訳集計)を実行できなかった。** 集計結果の代わりに、実行環境の
確認結果と、キーを用意すれば即実行できる手順を報告する。

## 確認したこと

1. **APIキーの有無**: `xc_client._load_api_key()` の解決順(Streamlit secrets →
   環境変数 `XC_API_KEY` → `xc_api_key.txt`)を1つずつ確認したが、いずれも
   未設定だった。
   - `xc_api_key.txt`: 存在しない
   - 環境変数 `XC_API_KEY`: 未設定
   - `.streamlit/secrets.toml`: 存在しない
   - `xc_client.is_enabled()` → `False`(実行して確認済み)

2. **xeno-canto自体への到達性(GloBIの403とは別問題であることの確認)**:
   キー無しで `https://xeno-canto.org/api/3/recordings` に直接アクセスしたところ、
   **HTTP 401**(`"Missing or invalid 'key' parameter"`)が返ってきた。
   これは「サービス自体には到達できているが、認証が必要」という正常な応答であり、
   `globi_client.py` が受ける組織ポリシーによる **403** ブロックとは別種の応答。
   つまり **xeno-cantoはこのサンドボックスからブロックされていない**。
   APIキーさえ手に入れば、この環境でも `license_audit.audit()` を実際に
   実行できる見込みが高い。

3. **対象種数**: `species_loader.BIRDS` は現在 **47種**(シードデータ使用、
   Sheets接続なしのためシードにフォールバック)。全種に `scientific`
   (学名)が設定済みで、監査対象から漏れる種はない。

## キーの入手方法(`xc_client.py` 冒頭コメント準拠)

1. https://xeno-canto.org/account/register で登録(無料)
2. https://xeno-canto.org/account/api-key でAPIキーを取得
3. 次のいずれかで設定:
   - ローカル開発: `toris_collection/xc_api_key.txt` にキーだけ1行で保存
     (`.gitignore` 済みなのでコミットされない)
   - 環境変数: `XC_API_KEY=<キー>`
   - Streamlit Cloud: secrets に `xc_api_key = "<キー>"`
4. キー設置後、`cd toris_collection && python check_xeno_canto_coverage.py`
   で音源カバレッジ、または以下でライセンス内訳を集計できる:

   ```python
   from species_loader import BIRDS
   from license_audit import audit, summarize
   rows = audit(BIRDS)
   print(summarize(rows))
   ```
