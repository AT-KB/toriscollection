# 企画部向け — 方針とルール

対象: 体験設計・機能仕様・数値バランス・ロードマップを担う人。

## 企画のミッション
「**受動的に開くたび、鳥のかわいさと癒しに出会い、遊ぶほど生態系がわかる**」を成立させる。
競争や作業感を持ち込まず、"よく会う友だちが増える"感覚を積み上げる。

## コア体験(壊してはいけない順序)
1. 離れる → 時間が進む(現実の時刻・季節)→ 鳥が来ている
2. 戻って発見 = 報酬(**鳴き声・ドット絵・落とし物**)
3. なぜ来たかが関係でわかる(食物網=学び)
4. 図鑑・みんなの庭・**会った日数**に静かに積もる → また来る理由

## 企画のルール(交渉不能)
- **急かさない/罰しない**。スタミナ・時短課金・デイリー強制・連続ボーナスの強要はしない。
  連続来訪の"ご褒美"は可、ただし**途切れても損しない**設計にする。
- **鳥の声と癒しは常に無料**。おやつ・課金は"多様性/意外性/彩り"にとどめ、アクセスを塞がない。
- **報酬の主役は鳥と鳴き声**。ポイントや通貨を主役にしない。
- **生態に誠実**。関係は GloBI ベース。「この植物→この虫→この鳥」を崩す恣意調整はしない。
- **庭が無い人でも楽しめる**。現実の庭・場所・道具を前提にした仕様は作らない。

## いま企画が持つ主要素
| 要素 | 状態 | 企画の論点 |
|---|---|---|
| バイオーム構築(植える/関係) | 実装済 | 「関係のプレビュー(植える→来る鳥)」を見せて学びを前に出す |
| ラジオ/BGM | 実装済 | 間・環境音・鳥の密度の心地よさ(耳の調整) |
| 会った日数(1日1回) | 実装済 | 節目(10/30/100)と称号、通知・図鑑での見せ方 |
| 落とし物(羽根/羽冠) | 実装済 | 出現率と"開封の喜び"の演出 |
| シミュ・ラボ(制約なし遷移) | 未 | 本編と分離、来た鳥はプレビュー(コレクションに入れない) |
| おやつ(=餌台ノード) | 連鎖ロジック済/UI未 | 無料資源で入手、レア来訪↑。gate しない |
| 収集→交換 | 未 | まず自前デジタル特典(記事/豆知識/レア声) |

## 次に企画で詰めること(優先度順)
1. **シミュ・ラボの仕様**: 制約なし・早送りで「遷移(草地→低木→二次林)」と来る鳥を見せる。
   ラボの鳥は"プレビュー"、本編の会った日数・図鑑には**入れない**(コレクションの価値を守る)。
2. **おやつの体験**: 「置く=関係を1本足す」。無料資源で入手、意外な鳥が来る楽しさ。課金は彩りのみ。
3. **通知の中身**(将来): 「🐦 ツバメが庭に来ています」等、受動再訪の一言。押しつけない頻度。
4. **数値バランス**: 到来確率/退去率/落とし物率/節目日数。指標は「1日1回で気持ちよく積もる」こと。

## 判断に迷ったら

1つでも No があれば作らない。
