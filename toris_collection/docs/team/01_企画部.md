# 企画部向け — 方針とルール

対象: 体験設計・機能仕様・数値バランス・ロードマップを担う人。

## 企画のミッション
「**受動的に開くたび、鳥のかわいさと癒しに出会い、遊ぶほど生態系がわかる**」を成立させる。
競争や作業感を持ち込まず、"よく会う友だちが増える"感覚を積み上げる。

## コア体験(壊してはいけない順序)
1. 離れる → 時間が進む(現実の時刻・季節)→ 鳥が来ている
2. 戻って発見 = 報酬(**鳴き声・ドット絵・落とし物**)
3. なぜ来たかが関係でわかる(食物網=学び)
4. 図鑑・みんなの庭・**会った日数**に静かに積もる → また来る理由

## 企画のルール(交渉不能)
- **急かさない/罰しない**。スタミナ・時短課金・デイリー強制・連続ボーナスの強要はしない。
  連続来訪の"ご褒美"は可、ただし**途切れても損しない**設計にする。
- **鳥の声と癒しは常に無料**。おやつ・課金は"多様性/意外性/彩り"にとどめ、アクセスを塞がない。
- **報酬の主役は鳥と鳴き声**。ポイントや通貨を主役にしない。
- **生態に誠実**。関係は GloBI ベース。「この植物→この虫→この鳥」を崩す恣意調整はしない。
- **庭が無い人でも楽しめる**。現実の庭・場所・道具を前提にした仕様は作らない。

## 次に企画で詰めること(2026-07-05改訂・優先度順)
1. **今回実装中の3点の詳細設計・監修**: バッジ演出/生態ログ/今日の庭の磨き込み
   (詳細: `docs/team/proposals/2026-07-05_核メカニクス強化企画提案.md`、
   `docs/team/proposals/2026-07-05_優先順位の妥当性検証.md`、
   `docs/team/proposals/2026-07-05_本物の生態系ビジョン深掘り.md`)。
2. **数値バランス**: 到来確率/退去率/落とし物率/節目日数。指標は「1日1回で気持ちよく積もる」こと。
   plume確率のfamiliarity底上げ(案は提案書参照)もここに含む。
3. **通知の中身**(将来・ネイティブアプリ化後): 「🐦 ツバメが庭に来ています」等、受動再訪の一言。
   押しつけない頻度。Web/TWAでは技術的に実現不可、Flutter本格化(Phase Y)後の検討事項。
4. **シミュ・ラボ/おやつ/収集→交換の中期部分**: いずれも棚上げ・不採用・後回し。
   再開する場合は本ドキュメントの改訂履歴および各提案書を参照して経緯を踏まえること。
