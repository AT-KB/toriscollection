# 社長向け — 現状の方針と経営判断ポイント

対象: 事業の方向・投資・意思決定を担う人。1枚で全体像。

## 何のプロダクトか(30秒)
**受動的な癒しアプリ。** 手のひらで小さな生態系を育てると、その関係から鳥が来て鳴く。
ユーザーは聴いて癒され、遊ぶほど生態系に詳しくなる。庭は要らない。
競合(ねこあつめ等の放置癒し)に対し、**本物の生態(GloBI)と本物の鳴き声**で差別化。

## 勝ち筋(なぜ伸びるか)
- **癒し × かわいい × 学び**の三点セットは希少。滞在時間(BGM)= 広告収益に直結。
- **習慣化装置**を内蔵(不在ループ+会った日数)。"よく会う友だち"が増えて再訪する。
- **実世界への興味**(バードウォッチング)へ橋渡しでき、コラボ/優待の収益余地が広い。

## ビジネスモデル(受動・低圧)
- **無料 + 控えめ広告**(静かなバナー + 任意リワード。全画面割り込みなし)。
- **小額サポーター課金**(広告オフ+バッジ)。ガチャ・スタミナ・時短はやらない方針。
- **中期**: バードウォッチング記事/優待のアフィリエイト、教育・保全コラボ(広告以外の柱)。
- 不変の一線: **鳥の声と癒しは常に無料**(ブランドと信頼の源泉)。

## 経営が握るべきリスクと判断
| 論点 | 状況 | 社長の判断 |
|---|---|---|
| **鳴き声の権利** | 広告=商用だと xeno-canto の非商用録音が使えない | 商用可(CC0/BY)に絞る方針でOKか。監査ツールは実装済 |
| **配信** | Web で開始可。Google Play は個人枠で「12人×14日テスト」規約。**2026-07-05: ローカル保存化によりバックエンド独立が不要になったため、TWA(Webの皮を被せる方式)を先行させる方針に変更** | 承認済み: TWAでのPlay進出(低コスト・低リスク)を先に進める。本格アプリ(Flutter・通知つき)は反応を見てから判断 |
| **スケール** | 個人データはローカル保存化(2026-07-04)によりGoogle Sheetsのスケール制約から外れた。Sheetsが残るのは将来「みんなの庭」を復活させる場合の匿名集計のみ | 当面DB移行は不要と判断。「みんなの庭」を本格的に伸ばす段階になった時にのみ再検討 |
| **収益 vs 世界観** | 課金は"彩り"限定=単価は低い | 広告+コラボで積む前提。短期の高収益化は狙わない合意 |
| **素材(ドット絵)** | 新種は既存絵を流用中 | 専用ドット絵の制作リソース(誰が/いつ) |

## 現在地(投資対効果)
- コストはごく小さい(Play登録 $25、Cloud/OSS は概ね無料枠)。
- プロダクトは**動くものが既にある**(構築/ラジオ/BGM/図鑑/会った日数/2バイオーム)。
- つまり「小さく出して改善を回す」段階。大型投資の前に**指標(継続率・滞在時間)**を取れる。

## 3〜6ヶ月の見立て(2026-07-05改訂)
1. Web(ローカル保存MVP)で公開 → 少人数で継続率・滞在時間・"会った日数"の伸びを観測。
2. Phase 0(Play開発者登録$25・プライバシーポリシー・PWAマニフェスト整備)を並行して低リスクで進める。
3. TWAでPlayへ(**DB移行を経由しない**。ローカル保存のままクローズドテスト12人×14日→製品版)。
   ここで初めて実際のDL数がPlay Consoleで見える。
4. 収益化はまずPlay Billing(軽い投げ銭/広告オフ課金)で試す。本格的な広告収益・通知は
   Flutterネイティブ化(Phase 2b、〜6ヶ月)の投資判断とセットで、反応を見てから決める。
5. 実世界コラボ(記事/優待)で"広告以外の収益とブランド"を育てる。

## 一言

静かな習慣を、広く長く届けることで収益化する事業。急がず、世界観を守って積む。
