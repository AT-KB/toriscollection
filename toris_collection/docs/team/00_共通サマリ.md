# #Toris Collection# — 全社共通サマリ(2026-06)

各部門ドキュメントの共通の土台。詳細思想は `CONCEPT_v2.md`(現在の正)、
経緯は `DIRECTION_BUSINESS_MODEL.md`、Play は `ROADMAP_GOOGLE_PLAY.md`。

## プロダクト一行
**受動的な癒しアプリ。手のひらで小さなバイオーム(生態系)を育てると、その関係から
鳥が現れて来て鳴く。聴いて癒され、いつのまにか生態系がわかる。** 庭は要らない。

## 3本の柱(KEY)
1. バイオーム構築のシミュレーション(遊びの核)
2. 声を聴く楽しみ(実在の鳴き声=報酬の主役)
3. 癒し(環境音+ときどきの鳥、BGMにできる)

## 交渉不能の原則(全部門共通のルール)
- **受動的**であること。急かす設計(スタミナ/時短課金/デイリー強制)は入れない。
- **罰しない**。撹乱で庭が痩せてもコレクションは減らない。
- **鳥の声と癒しは常に無料**。「課金しないと聴けない」は絶対にやらない。
- **生態に誠実**。関係(食物網)は GloBI ベース。恣意的な"なぜ来たか"を足さない。
- **かわいさ**(来る瞬間・鳴き声・ドット絵)を最優先に守る。

## 現状(実装済み / 進行中)
- 実装済み: バイオーム構築、ラジオ(実時刻/季節/発声スケジューラ)、ヒーリングBGM、
  図鑑・落とし物、みんなの庭、不在ループ、**会った日数(日次カウント)**、
  京都/シャーロットの種拡張、餌台連鎖ロジック(UI未配線)、ライセンス監査。
- 収益前提: 広告=商用 → 鳴き声は商用可(CC0/BY)に絞る(監査ツール実装済・切替は判断待ち)。

## 用語
- **バイオーム**: ユーザーが育てる仮想の生態系(京都/シャーロット等)。
- **不在ループ**: 離れている間に時間が進み鳥が来る仕組み。受動性の核。
- **落とし物(羽根/羽冠)**: 留守中の小さな報酬。
- **みんなの庭**: 匿名の集合地図(誰の庭に何が来ているか)。
