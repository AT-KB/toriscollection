# Toris Collection ドット絵デザインガイド

ポケモン ルビー・サファイア・エメラルド時代のスプライト品質を目指す、35鳥分のドット絵制作ガイド。

---

## 0. 仕様サマリ

| 項目 | 値 |
|---|---|
| サイズ | 64×64 ピクセル |
| フォーマット | PNG(透過対応) |
| 色数 | 各鳥 16〜32色程度(パレット制限風) |
| 配置 | `sprites/birds/{bird_id}.png` |
| 拡大表示 | CSS の `image-rendering:pixelated` で滑らかにせずピクセルを保つ |

ファイル名は `data.py` の `BIRDS` の各 ID と一致させる(例: `shijukara.png`、`mejiro.png`)。

---

## 1. スタイル指針

### ポーズと角度

- **真横向き or 斜め前 4/5**(ポケモン GBA 時代の正面寄り斜め)
- **止まっている姿勢**(羽ばたき中ではなく、枝に止まっているような落ち着いた姿勢)
- 体は 64×64 の中央付近、尾が画面端ギリギリにかかるくらいの大きさ

### キャラクター性

- **写実より特徴抽出**: 学術図鑑ではなくゲームの図鑑なので、判別性最優先
  - 冠羽がある鳥(キバタン、ショウジョウコウカンチョウ等)は冠羽を大きく
  - 嘴の特徴的な形(カワセミの長い嘴、ハチドリの細い嘴)は誇張
  - 体の色のコントラストを強める
- **目に光を入れる**: 1〜2ピクセルの白ハイライトで生命感を出す
- **羽毛の縞模様や斑点**は意図的なドット配置で再現

### 配色

各鳥のメインカラーは `data.py` の `color` フィールドを参照:

```python
"color": "#5a8a5a",  # 例: シジュウカラ
```

これを基準に、明度差で立体感を作る:
- 明色(ハイライト)
- 中間色(基本色)
- 暗色(影)
- アクセント色(特徴的な部位)

合計 4〜8色のパレットで構成すると、ドット絵らしい統一感が出る。

## 2. 制作の優先順位

### Phase 1: 主要バイオームの代表3種ずつ(計9種)

各バイオームの「象徴的な鳥」を最初に作る:

**京都**:
- シジュウカラ(`shijukara`)
- メジロ(`mejiro`)
- ウグイス(`uguisu`)

**シドニー**:
- ワライカワセミ(`kookaburra`)
- ゴシキセイガイインコ(`rainbow_lorikeet`)
- アオアズマヤドリ(`satin_bowerbird`)

**シャーロット**:
- ショウジョウコウカンチョウ(`northern_cardinal`)
- アオカケス(`blue_jay`)
- ルビーノドハチドリ(`ruby_throated_hummingbird`)

これでテスターに「ドット絵の方向性」を確認してもらえる。

### Phase 2: PLUME_BIRDS(隠しレア鳥9種)

`mementos.py` の `PLUME_BIRDS` に該当する鳥は、ドット絵の見栄えが特に重要(羽冠が落とし物の目玉)。Phase 1で重複していない鳥を追加:

- キバタン
- エボシクマゲラ
- キビタキ
- カワセミ
- イカル
- キバラオーストラリアコマドリ

### Phase 3: 残りの全鳥(約20種)

Phase 1-2 でスタイルが固まったら、機械的に残りを描く。1日2〜3種ペースで進めれば1〜2週間で完成。

## 3. 制作ツールの推奨

### 選択肢A: Aseprite($20、最高品質)

ドット絵専用ツールの定番。パレット管理、レイヤー、アニメ機能あり。本格的にやるなら最適。
https://www.aseprite.org/

### 選択肢B: Piskel(無料、ブラウザ完結)

ブラウザで動く無料のドット絵エディタ。気軽に試せる。
https://www.piskelapp.com/

### 選択肢C: AI 生成 + 手直し

参考プロンプト(Stable Diffusion / DALL-E等):
> "Pixel art sprite of a Japanese tit bird, 64x64 pixels, side view, GBA Pokemon style, green and white plumage, transparent background, limited color palette"

ただしAI生成は「ピクセル単位の意図」を理解しないため、最終調整は手作業が必要。

### 選択肢D: ココナラ等で外注

1鳥 500〜2000円程度で外注も可能。35鳥で2〜7万円。クオリティ重視ならアリ。

## 4. 動作確認方法

サンプルとしてシジュウカラ・メジロの簡易ドット絵を `sprites/birds/` に配置済み(動作確認用、本番では描き直し前提)。

ローカルで `py -m streamlit run app.py` を実行し、図鑑タブでシジュウカラまたはメジロを開くと、スプライトが大きく表示されるはず。

ファイルがない鳥は自動的に Emoji(🐦)にフォールバックする。

## 6. テスト時の確認

- [ ] スプライトが図鑑詳細で 128×128 px で表示される

- [ ] スプライトがない鳥は Emoji にフォールバックする
- [ ] ファイルサイズが大きすぎない(各 5KB 以下が目安)

## 7. 将来の拡張

### アニメーション(後の課題)

各鳥に2〜3フレームのアニメ(瞬き、羽ばたき)を加えると更にゲーム感が増す。GIFまたはCSSスプライトシートで実装可能。
ただし制作労力が3倍になるので、まずは静止画で完成させてから検討。

### 落とし物のドット絵化

羽根・小枝・羽冠も各鳥固有のドット絵にできる。35鳥 × 3カテゴリ = 105 種類。
ただし、すでに「鳥の色を反映した汎用アイコン」で表示できているので、優先度は低い。

### フィールド表示への適用

ホーム画面のフィールドSVGで、Emoji の代わりにドット絵を表示する。
これも実装可能だが、SVGに data URL を埋め込む形になり、ファイルサイズが増える(各鳥分のPNGを毎回送信)。
**まずは図鑑のみでスタート**、好評ならフィールドにも展開する方針。
