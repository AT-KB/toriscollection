## ゴール

**「鳥のかわいさ」を広めるスマホアプリとして Google Play で公開する。**
コアの魅力(ユーザー調査 = 開発者自身の実感):

- みんなの庭に鳥が来る、その瞬間がかわいい
- 鳥の鳴き声がかわいい
- ドット絵の鳥がかわいい

この3つを**開いた瞬間に体験できる**ことをアプリの背骨にする。

## 1. 現状と、Play に出すうえでの5つの壁

現構成(2026-07-05時点): Streamlit + **ローカル保存(セーブコード方式、個人データの主経路)**
+ Google Sheets(ベストエフォート・将来の「みんなの庭」用に残置) + xeno-canto(音声を base64 で埋め込み) + Streamlit Cloud

1. **Streamlit はそのままアプリにならない**
   TWA(Trusted Web Activity)で包めば Play に出せるが、PWA 品質(manifest / service worker / HTTPS)が必要。オフライン不可・プッシュ通知が実質使えない・起動が重い、はこの方式の限界として受け入れる。
2. **API キーは端末に置けない**
   今は Streamlit secrets で安全だが、ネイティブアプリに埋めると抽出される。**音声はサーバー経由(プロキシ+キャッシュ)**にする必要がある。
3. **xeno-canto のライセンス**
   録音の多くは CC BY-NC-SA / BY-NC-ND。**無料アプリ + 録音者クレジット表記**なら現実的。商用化するなら CC BY / CC0 の録音だけに絞るフィルタが必要(コードには既に citation 取得がある。ライセンス列の集計を足す)。
4. ~~**Google Sheets はスケールしない**~~ → **2026-07-05: 個人データについては解消**。
   レート制限(読み取り 300req/分/プロジェクト程度)は元々の懸念だったが、2026-07-04の
   ローカル保存化(セーブコード方式)で個人の進行データはSheetsを経由しなくなった。
   残る懸念は「みんなの庭」の匿名集計だけ(現在MVPでは非表示)で、これを本格的に
   伸ばす段階になって初めてFirestore/Supabase移行を再検討すればよい。
5. **Google Play の規約**
   個人デベロッパーアカウントは **「12人のテスターで14日間のクローズドテスト」を経ないと製品版公開できない**。登録料 $25(買い切り)。この期間を逆算して早めにテストを始める。

## 2. フェーズ計画(2026-07-05改訂: バックエンド独立を前提から外し、TWA先行に組み替え)

### Phase 0: 土台(今〜2週間)
- [ ] Google Play デベロッパーアカウント登録($25)— 審査に数日かかるので最初に
- [ ] ライセンス監査スクリプト: 使用中の全録音の CC 種別を集計(`check_xeno_canto_coverage.py` を拡張)
- [ ] プライバシーポリシーページ(静的1枚。Play 提出に必須。**ローカル保存化により
      「サーバーに個人データを保存しない」と書ける分、内容はシンプルになる**)
- [ ] 上の「決めごと」4点を確定

### Phase 1: TWA版をPlayへ(旧Phase2a、バックエンド独立を待たずに着手)
2026-07-04のローカル保存化により、この段階で新規サーバーは不要。

- [ ] Streamlit 側に manifest + アイコンを整備(PWA 化は最小限)
- [ ] Bubblewrap で TWA 生成 → 内部テスト → **クローズドテスト(12人×14日)を開始**
  - 今のテスター仲間に Play 経由で入ってもらう = 規約消化と実地テストを兼ねる
- [ ] ストア素材: アイコン(512px)、スクショ、フィーチャーグラフィック、説明文
- [ ] データセーフティフォーム、コンテンツレーティング(全年齢。「サーバーに個人データを
      保存しない」ため記入もシンプルになる見込み)

### Phase 2: 収益化を試す(Play Billing、新設)
「一通りのアプリ開発フロー(公開・収益化・DL数を見る)をやり切る」ため、
広告SDKより先に、新規サーバー不要で組み込める軽量な課金を試す。

- [ ] Play Billing で「広告オフ/投げ銭」程度の軽いサポーター課金を1つ実装
- [ ] 反応を見て、広告(要 `xc_client.COMMERCIAL_ONLY=true` 化・使用可能録音の絞り込み)を
      追加するかはPhase 3以降で判断

### Phase 3: 製品版公開
- [ ] クローズドテストの知見反映 → 製品版申請
- [ ] クレジット画面: 録音者(xeno-canto)・Freesound・使用ライブラリ

### Phase 4: かわいさ拡散(公開後の成長ループ)
- [ ] **共有カード**: 「今日、ツバメが庭に来ました」+ ドット絵 + 鳴き声QR を1枚画像で自動生成 → X/Instagram へ
- [ ] 15秒の鳴き声つき動画エクスポート(将来)
- [ ] ASO: 「鳥 放置ゲーム かわいい 鳴き声 癒し」等で検索最適化

---

### Phase X(保留・将来判断): バックエンド独立(旧Phase1)
**Play進出の前提条件ではなくなった**。継続率・DL数の反応が良く、「みんなの庭」を
本格的に伸ばす、または通知つきネイティブ化(Phase Y)に進む段階になって初めて着手を判断する。

- [ ] FastAPI サーバー(Cloud Run、無料枠)
  - `GET /garden/{user}` 庭の状態 / `POST /plant` / 不在ループの実行
  - `GET /audio/{species}` 音声プロキシ: xeno-canto をサーバーでキャッシュし **URL 配信**(base64 埋め込み廃止 → Web 版の読み込みも速くなる)
  - ロジック(engine.py / disturbance.py / absence_loop.py)はそのまま Python で活きる
- [ ] DB 移行: Sheets(みんなの庭用の匿名集計のみ) → **Firestore**(または Supabase)
- [ ] Streamlit 版をこの API に乗せ替え

### Phase Y(保留・将来判断): Flutter 本格アプリ(旧Phase2b)
かわいさを最大化する専用クライアント。反応が良い場合のみ着手判断(強制タイムラインなし)。
3画面に絞る:

1. **ラジオ**(ホーム): 鳴き声スケジューラを Dart に移植。ドット絵がとまり木で揺れ、鳴くとき ♪ が出る
2. **庭**: 鳥が来る瞬間のアニメーション(飛来 → 着地 → 鳴く)。ここが「かわいい」の主戦場
3. **図鑑**: 集めた鳥・鳴き声・落とし物

- [ ] **プッシュ通知(FCM)**: 「🐦 ツバメが庭に来ました」— 不在ループとの相性が最高で、再訪の核になる
- [ ] 鳴き声はネイティブ再生(just_audio)+ サーバーキャッシュ済み音源(Phase Xのバックエンド独立が前提)
- [ ] 本格的な広告(AdMob等)収益化はこの段階で検討(TWAでは相性が弱いため)
- [ ] TWA 版と同じアプリID で**置き換え更新**(ユーザーはそのままアップデートで移行)

## 3. コスト見積り

| 項目 | 費用 |
|---|---|
| Google Play 登録 | $25(1回のみ) |
| Cloud Run + Firestore | **Phase X(保留)まで発生しない**。ローカル保存MVPのままなら追加サーバー費ゼロ |
| Flutter / FastAPI | 無料(OSS)。**Phase Y/X着手時のみ必要** |
| 音源・ドット絵 | 手持ち資産(designbird/)+ xeno-canto(クレジット表記) |

## 4. リスク

- **xeno-canto 規約・API 変更** → サーバーキャッシュで影響を緩和。長期的には CC0 音源の比率を上げる
- **個人開発の工数** → フェーズを小さく切り、各フェーズ末に「動くもの」を出す。Claude Code をフル活用
- **TWA 版の体験が低評価を招く** → クローズドテスト限定に留め、製品版レビューが立つのは Flutter 版から、という選択肢も残す
- **TWAでは広告収益化が弱い**(2026-07-05追記) → Phase 2ではPlay Billingの軽量課金に留め、
  本格的な広告収益はPhase Y(Flutterネイティブ化)まで待つ前提を明記しておく
- **ローカル保存ゆえの複数端末非対応**(2026-07-05追記) → TWA版でも「この端末にのみ保存」
  という制約は変わらない。クローズドテスト参加者への案内文に明記する
