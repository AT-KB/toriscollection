# Google Play 公開ロードマップ — #Toris Collection#

作成: 2026-06-14 / 前提が変わったら随時更新する

## ゴール

**「鳥のかわいさ」を広めるスマホアプリとして Google Play で公開する。**
コアの魅力(ユーザー調査 = 開発者自身の実感):

- みんなの庭に鳥が来る、その瞬間がかわいい
- 鳥の鳴き声がかわいい
- ドット絵の鳥がかわいい

この3つを**開いた瞬間に体験できる**ことをアプリの背骨にする。

## 0. 最初に決めること(未確定の分岐)

| # | 決めごと | 推奨(仮の前提) | 理由 |
|---|---|---|---|
| 1 | 配信方式 | **段階式**: まず Web を TWA で包んで早期に Play へ → 並行して Flutter 本格アプリに置き換え | 早く出して規約(下記テスト義務)も先に消化。最終体験も妥協しない |

| 3 | かわいさの広め方 | **アプリ内演出 + SNS共有の両輪** | 通知・演出で「開く喜び」、共有カードで「外へ広がる」 |
| 4 | 時期 | 3ヶ月で TWA 版、6〜9ヶ月で Flutter 版 | 個人開発 + Claude Code 支援の現実的ペース |

## 1. 現状と、Play に出すうえでの5つの壁

現構成: Streamlit + Google Sheets + xeno-canto(音声を base64 で埋め込み) + Streamlit Cloud

1. **Streamlit はそのままアプリにならない**
   TWA(Trusted Web Activity)で包めば Play に出せるが、PWA 品質(manifest / service worker / HTTPS)が必要。オフライン不可・プッシュ通知が実質使えない・起動が重い、はこの方式の限界として受け入れる。
2. **API キーは端末に置けない**
   今は Streamlit secrets で安全だが、ネイティブアプリに埋めると抽出される。**音声はサーバー経由(プロキシ+キャッシュ)**にする必要がある。
3. **xeno-canto のライセンス**
   録音の多くは CC BY-NC-SA / BY-NC-ND。**無料アプリ + 録音者クレジット表記**なら現実的。商用化するなら CC BY / CC0 の録音だけに絞るフィルタが必要(コードには既に citation 取得がある。ライセンス列の集計を足す)。
4. **Google Sheets はスケールしない**
   レート制限(読み取り 300req/分/プロジェクト程度)があり、ユーザーが数十人を超えると破綻する。Firestore か Supabase へ移行する。
5. **Google Play の規約**
   個人デベロッパーアカウントは **「12人のテスターで14日間のクローズドテスト」を経ないと製品版公開できない**。登録料 $25(買い切り)。この期間を逆算して早めにテストを始める。

## 2. フェーズ計画

### Phase 0: 土台(今〜2週間)
- [ ] Google Play デベロッパーアカウント登録($25)— 審査に数日かかるので最初に
- [ ] ライセンス監査スクリプト: 使用中の全録音の CC 種別を集計(`check_xeno_canto_coverage.py` を拡張)
- [ ] プライバシーポリシーページ(静的1枚。Play 提出に必須)
- [ ] 上の「決めごと」4点を確定

### Phase 1: バックエンド独立(〜1.5ヶ月)
Web 版・アプリ版の両方が使う「共通の脳」を作る。

- [ ] FastAPI サーバー(Cloud Run、無料枠)
  - `GET /garden/{user}` 庭の状態 / `POST /plant` / 不在ループの実行
  - `GET /audio/{species}` 音声プロキシ: xeno-canto をサーバーでキャッシュし **URL 配信**(base64 埋め込み廃止 → Web 版の読み込みも速くなる)
  - ロジック(engine.py / disturbance.py / absence_loop.py)はそのまま Python で活きる
- [ ] DB 移行: Sheets → **Firestore**(または Supabase)。移行スクリプトで既存テスターのデータを引き継ぐ
- [ ] Streamlit 版をこの API に乗せ替え(= Web 版が最初の API クライアント)

### Phase 2a: TWA 版を Play へ(Phase 1 と並行、〜2ヶ月)
- [ ] Streamlit 側に manifest + アイコンを整備(PWA 化は最小限)
- [ ] Bubblewrap で TWA 生成 → 内部テスト → **クローズドテスト(12人×14日)を開始**
  - 今のテスター仲間に Play 経由で入ってもらう = 規約消化と実地テストを兼ねる
- [ ] ストア素材: アイコン(512px)、スクショ、フィーチャーグラフィック、説明文
- [ ] データセーフティフォーム、コンテンツレーティング(全年齢)

### Phase 2b: Flutter 本格アプリ(〜6ヶ月)
かわいさを最大化する専用クライアント。3画面に絞る:

1. **ラジオ**(ホーム): 鳴き声スケジューラを Dart に移植。ドット絵がとまり木で揺れ、鳴くとき ♪ が出る
2. **庭**: 鳥が来る瞬間のアニメーション(飛来 → 着地 → 鳴く)。ここが「かわいい」の主戦場
3. **図鑑**: 集めた鳥・鳴き声・落とし物

- [ ] **プッシュ通知(FCM)**: 「🐦 ツバメが庭に来ました」— 不在ループとの相性が最高で、再訪の核になる
- [ ] 鳴き声はネイティブ再生(just_audio)+ サーバーキャッシュ済み音源
- [ ] TWA 版と同じアプリID で**置き換え更新**(ユーザーはそのままアップデートで移行)

### Phase 3: 製品版公開
- [ ] クローズドテストの知見反映 → 製品版申請
- [ ] クレジット画面: 録音者(xeno-canto)・Freesound・使用ライブラリ

### Phase 4: かわいさ拡散(公開後の成長ループ)
- [ ] **共有カード**: 「今日、ツバメが庭に来ました」+ ドット絵 + 鳴き声QR を1枚画像で自動生成 → X/Instagram へ
- [ ] みんなの庭の世界地図を強化(誰かの庭に今日来た鳥が流れる)
- [ ] 15秒の鳴き声つき動画エクスポート(将来)
- [ ] ASO: 「鳥 放置ゲーム かわいい 鳴き声 癒し」等で検索最適化

## 4. リスク

- **xeno-canto 規約・API 変更** → サーバーキャッシュで影響を緩和。長期的には CC0 音源の比率を上げる
- **個人開発の工数** → フェーズを小さく切り、各フェーズ末に「動くもの」を出す。Claude Code をフル活用
- **TWA 版の体験が低評価を招く** → クローズドテスト限定に留め、製品版レビューが立つのは Flutter 版から、という選択肢も残す
