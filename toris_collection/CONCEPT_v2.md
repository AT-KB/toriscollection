# コンセプト再定義 v2 — #Toris Collection#

2026-06 時点の確定思想。DIRECTION_BUSINESS_MODEL.md の散らばった議論を、この1枚に集約する。
(細部の根拠は同ファイルを参照。ここが"現在の正"。追加構想は末尾 §拡張構想 v2.1。)

## 一行

**受動的な癒しアプリ。小さな生態系(バイオーム)を育てると、その関係から鳥が現れて
やって来て鳴く。あなたは聴いて、癒されて、いつのまにか生態系がわかるようになる。**

庭は要らない。手のひらの中で育てる小さな世界。放っておくと時間が進み、鳥が来ている。

## 3本の柱(KEY)

1. **バイオーム構築のシミュレーション** — 何を植え、どんな関係を作るかで、現れる生き物が変わる。
   これが"おもちゃ"の本体。生態のつながりを組むこと自体が遊び。
2. **声を聴く楽しみ** — 実在の鳥の鳴き声(xeno-canto)。来た鳥が本当に鳴く。
3. **癒し** — 環境音+ときどきの鳥。BGMとして流しっぱなしにできる。急かさない・罰しない。

この3つを **開いた瞬間に** 感じられることを設計の背骨にする。

## 「餌箱→リス避け」を、関係性の構築に置き換える(考え直した核)

アメリカの裏庭は「餌台を置いてリスを避け、狙った鳥を呼ぶ」。この面白さの本質は
**"何を差し出すかで、来るものが変わる"** という因果。これを害獣退治のミニゲームにするのではなく、
**バイオーム=生態的な関係を組むこと**そのものに置き換える。

- プレイヤーは「植物→昆虫→鳥」「実→果実食の鳥」「種子→種食の鳥」などの**関係を設計**する。
- 組み合わせによって現れる顔ぶれが変わり、**「なぜこの鳥が来たか」が関係で説明される**(GloBI由来・既存)。
- リス・猛禽などの動物連鎖(`feeder_chain.py` で実装済み)は、**関係の一種**として後から足す
  "スパイス"であって、主役ではない。主役はあくまで **バイオームのつながりを読む楽しさ**。

→ 狙いは「攻略 = 生態を理解すること」。遊ぶほど食物網の読み方が身につく = 生態系知識が深まる。

## 庭がない人でも楽しめる(アクセシビリティ)

- 育てるのは**仮想のバイオーム**。現実の庭・土・道具は一切要らない。誰でも手ぶらで始められる。
- 場所を選ばない受動性: 通勤・作業・就寝のお供に、開けば世界が進んでいる。
- (任意・低コスト)地域プール: 望めば「あなたの地域に実在する種」で遊べる余地は残す
  (必須にはしない。天気連動のような重い日常連動はやらない方針は維持)。

## 受動のコアループ

```
 離れる → 時間が進む → 関係から鳥が現れて来ている
   → 戻って「誰が来たか」を発見(声・ドット絵・落とし物 = 癒しと収集の報酬)
     → なぜ来たかが関係でわかる(学び)
       → 図鑑・みんなの庭に増える → また来る理由(季節・通知)
```

急かすもの(スタミナ/時短課金/デイリー強制)は入れない。**戻る理由は"かわいさと癒し"だけ**。

## 機能の再定義(優先度つき)

| 優先 | 機能 | 役割 | 状態 |
|---|---|---|---|
| ★★★ | バイオーム構築(植える・関係を組む) | 遊びの核 | あり(京都/シャーロット拡張済) |
| ★★★ | ラジオ(来た鳥の実声) | 報酬・癒し | あり(実時刻/季節/スケジューラ済) |
| ★★★ | ヒーリングBGMモード | 滞在時間=癒し=広告リターン | あり(実装済) |
| ★★☆ | 図鑑・落とし物(収集) | 進行・愛着 | あり(羽冠率UP済) |
| ★★☆ | 不在ループ + 通知 | 受動再訪 | ループあり/通知は将来(PWA/アプリ) |
| ★★☆ | 「なぜ来たか」= 生態の可視化 | 学び | あり(GloBI由来) |
| ★☆☆ | みんなの庭(集合地図) | 共感・拡散 | あり |
| ★☆☆ | 動物連鎖(リス/猛禽) | 関係の"スパイス" | ロジック実装済・UI未配線 |

**やらない/後回し**: 天気連動、害獣退治ミニゲーム化、ガチャ・重い課金、歩数連動(Web不安定)。

## ビジネスモデル(受動・低圧・広告で副収入)

- **無料 + 控えめ広告**: 静かなバナー1枚 + 任意のリワード広告(「見ると珍しい鳥が来やすい」)。
  全画面割り込みは無し。
- **サポーター課金(1回・小額)**: 広告オフ + バッジ + 図鑑フル。ガチャはしない。
- **前提の制約**: 広告=商用 → xeno-canto の NC 録音は使えない → **商用可(CC0/BY/BY-SA/BY-ND)に絞る**
  (`COMMERCIAL_ONLY` フラグと監査ツールは実装済。デプロイで監査→ONの順)。
- 将来: 教育コラボ(野鳥の会・博物館)= 広告以外の収入と信頼。「生態が学べる」強みが効く。

## 差別化(なぜ他の放置ゲームでない)

- ねこあつめ的な"置いて待つ癒し"に、**本物の生態(GloBIの食物網)と本物の鳴き声**が乗る。
- 攻略が"生態の理解"になり、遊ぶほど賢くなる。かわいい+癒し+学びの三点セットは希少。

## 直近の道筋(今の実装をベースに)

1. デプロイして新種(京都/シャーロット)と BGM・羽冠率を体感確認。
2. GloBI プレビュー(開発パネル)で新種の食性を裏取り → ズレを微調整。
3. ライセンス監査 → `commercial_only` を判断。
4. 準備が整えば #2(動物連鎖)を"関係のスパイス"として最小配線。
5. 中期: PWA 通知 → TWA で Google Play(規約: 12人×14日テスト)。

### 芯: 「おやつ = 餌台 = 食物網のノード(関係)」(過去の構想・現在は不採用)
ねこあつめの"アイテム"と、本アプリの"バイオーム構築"は**同じもの**にできる、という
発想でおやつ機能を構想していたが、上記2026-07-05改訂により不採用となった。
以下は経緯を残すための記録(実装しない)。
おやつ(木の実ケーキ・花蜜・スエット等)は、生態的には**餌台=関係のノード**。
→ 課金/収集要素を生態エンジン(既存 `feeder_chain.py`)の上に自然に乗せられる、
   という考えだった。「アイテムを置く」= 「関係を1本足す」= 「来る鳥が変わる」。

### 2つのモード(本編を核に、実験レイヤーは棚上げ中)
1. **本編(受動・コレクション)** — 現実的制約(土地・気温・時間)あり。放置で鳥が来て鳴く。
   ここが"癒しと習慣"の本体。**報酬の主役は鳥と鳴き声**。MVPではここに集中する。
2. **シミュレーション・ラボ(制約なし・実験)** — **2026-07-05・CEO判断によりMVPでは棚上げ**
   (仕様検討も含め今は着手しない)。将来、余力ができた段階で「土地/種苗コスト/時間を
   取り払い、何を組むとどんなバイオームに遷移し、どんな鳥が来るかを早送りで試せる」学びの場
   として再検討する。ここで来た鳥は"プレビュー"に留め、本編コレクション/日次カウントには
   入れない(=コレクションの価値を薄めない)という設計方針自体は将来のために残す。

### 日次カウント(習慣化 → 本物の鳥への興味)
- **鳥ごとに「来たのべ日数」を累計**。増分は **1日1回まで**(グラインド不可・受動的で穏やか)。
  既存の `observed[bird]["count"]` を「1日1カウント」に整える(last_counted_date を持つ)。
- 表示: 図鑑に「◯◯に会った日数 12日」。節目(10/30/100日)で静かな称号。
- 狙い: 競争でなく**"よく会う友だち"が増える感覚**。北極星は「現実の鳥に興味が向く」。
  将来は「あなたのよく会う鳥の実在情報/保全リンク」へ、押しつけずに橋渡し(募金の強制はしない)。

### 収集→称号・バッジ(旧「収集→交換」、2026-07-05に改称・第一歩に縮小)
- **第一歩(今回実装)**: 節目(10/30/100日)到達時に、図鑑上で静かなバッジ演出を表示する。
  既存の`load_bird_days`データのみで完結し、新規データ追加はゼロ。
- **生態ログ(2026-07-05新規追加)**: `absence_loop.build_reason_text()`が生成する
  「なぜ来たか」を、これまで揮発表示のみだったものを図鑑に蓄積表示する。
  「あなたが組んだ関係(食物網)が、実際に鳥を呼んだという証拠」として位置づける
  (詳細: `docs/team/proposals/2026-07-05_核メカニクス強化企画提案.md`案2、
  `docs/team/proposals/2026-07-05_本物の生態系ビジョン深掘り.md`§4案2)。
- **後回し**: デジタル特典(豆知識カード等)の解錠は今回スコープ外、余力があれば着手。
- **中期(将来検討、着手判断は別途)**: 光学機器・図鑑・保全団体とのアフィリエイト/優待、

### 実装の当たり(小さく出す順、2026-07-05改訂)

   「会った異なる暦日」を集計(=1日1カウント)し、図鑑に「会った日数 N日」+

3. **生態ログ**(今回実装): 「なぜ来たか」の蓄積表示。
4. **今日の庭の磨き込み**(今回実装、①と並走可): ドット絵サムネイル・鳴き声プレビューを追加。
